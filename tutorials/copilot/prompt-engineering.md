# Copilot 提示词技巧

参考来源：[GitHub Docs 入门](https://docs.github.com/zh/get-started)。

## 写好提示词的结构

```text
背景：这个项目使用 React + Vite。
目标：实现一个可搜索的仓库列表。
约束：不要引入新依赖，保持移动端可用。
输出：给出需要修改的文件和关键代码。
```

## 提供上下文

Copilot 更擅长基于已有代码工作。提问时给出：

- 当前文件或函数名
- 预期行为
- 错误信息
- 已尝试方案
- 不希望改变的边界

## 常见任务

| 任务 | 提示方式 |
| --- | --- |
| 解释代码 | “解释这个函数如何处理异常路径” |
| 写测试 | “为这个工具函数补边界测试” |
| 重构 | “保持行为不变，降低重复逻辑” |
| 修 bug | “根据报错定位可能原因，并给最小修改” |
